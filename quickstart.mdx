## Installation

Get TwiP running locally in three steps:

<Steps>
  <Step title="Clone the repository">
    Download the TwiP source code from GitHub:
    
    ```bash
    git clone https://github.com/anirudhtulasi/TwiP.git
    cd TwiP
    ```
  </Step>
  
  <Step title="Install dependencies">
    Install all required Python packages using pip:
    
    ```bash
    pip install -r requirements.txt
    ```
    
    This installs Flask, Tweepy, scikit-learn, NLTK, pandas, and other dependencies.
  </Step>
  
  <Step title="Configure Twitter API credentials">
    Open `predict.py` and add your Twitter API credentials:
    
    ```python predict.py
    ckey = 'YOUR_CONSUMER_KEY'
    csecret = 'YOUR_CONSUMER_SECRET'
    atoken = 'YOUR_ACCESS_TOKEN'
    asecret = 'YOUR_ACCESS_SECRET'
    ```
    
    Get credentials by creating a Twitter Developer account at [developer.twitter.com](https://developer.twitter.com).
  </Step>
</Steps>

## Run the application

Start the Flask development server:

<CodeGroup>
```bash Python 3
python3 application.py
```

```bash Python
python application.py
```
</CodeGroup>

The server starts on `http://localhost:8000` with debug mode enabled.

## Analyze your first user

Once the server is running, you can analyze any Twitter user:

<Steps>
  <Step title="Open the web interface">
    Navigate to `http://localhost:8000` in your browser. You'll see the TwiP interface with a username input field.
  </Step>
  
  <Step title="Enter a Twitter username">
    Type a Twitter handle (without the @ symbol) into the text field. For example: `elonmusk` or `BillGates`.
  </Step>
  
  <Step title="View personality results">
    Click the "Check" button. TwiP will:
    
    - Fetch up to 4,000 recent tweets from the user
    - Preprocess and analyze the text
    - Display the four-letter MBTI type (e.g., INTJ)
    - Show the personality archetype (e.g., "The Mastermind")
    - Provide a detailed personality description
  </Step>
</Steps>

## How it works

When you submit a username, TwiP executes the following pipeline:

### 1. Tweet collection

The `getTweets()` function in `predict.py:116` uses the Tweepy API to fetch tweets:

```python predict.py
def getTweets(user):
    csvFile = open('user.csv', 'a', newline='')
    csvWriter = csv.writer(csvFile)
    try:
        for i in range(0, 4):
            tweets = api.user_timeline(
                screen_name=user, count=1000, include_rts=True, page=i)
            for status in tweets:
                tw = preproc(status.text)
                if tw.find(" ") == -1:
                    tw = "blank"
                csvWriter.writerow([tw])
    except tweepy.TweepError:
        print("Failed to run the command on that user, Skipping...")
    csvFile.close()
```

### 2. Text preprocessing

The `preproc()` function at `predict.py:79` applies:

- Tokenization with custom regex patterns
- Stopword removal using NLTK
- Stemming with SnowballStemmer
- Removal of URLs, mentions, and numbers
- Unicode normalization

### 3. Classification

The `classify_user()` function at `predict.py:132` loads four pre-trained Naive Bayes models:

```python predict.py
model_IE = pickle.load(open("Pickle_Data/BNIEFinal.sav", 'rb'))
model_SN = pickle.load(open("Pickle_Data/BNSNFinal.sav", 'rb'))
model_TF = pickle.load(open('Pickle_Data/BNTFFinal.sav', 'rb'))
model_PJ = pickle.load(open('Pickle_Data/BNPJFinal.sav', 'rb'))
```

Each model predicts one MBTI dimension, and results are combined using majority voting.

### 4. Flask routing

The main route in `application.py:6` handles both GET and POST requests:

```python application.py
@app.route("/",methods=["GET","POST"])
def home():
    if request.method == 'POST':
        uid = request.form.get("userid")
        name,mbti,personality = classify_user(uid)
        return render_template('analysis.html',display=True,name=name,mbti=mbti,personality=personality)
    return render_template('analysis.html',display=False)
```
