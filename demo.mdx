## Understanding your results

TwiP analyzes recent tweets to determine the Myers-Briggs personality type across four dimensions:

### Introversion (I) vs Extraversion (E)

Determines whether the user's language patterns suggest:
- **Introversion**: Focus on internal thoughts, reflection, and solitary activities
- **Extraversion**: Emphasis on external interaction, social engagement, and group dynamics

### Sensing (S) vs Intuition (N)

Analyzes how the user processes information:
- **Sensing**: Concrete details, facts, and present realities
- **Intuition**: Abstract concepts, future possibilities, and patterns

### Thinking (T) vs Feeling (F)

Evaluates decision-making style:
- **Thinking**: Logic, objectivity, and analytical reasoning
- **Feeling**: Empathy, values, and emotional considerations

### Perceiving (P) vs Judging (J)

Assesses approach to structure:
- **Perceiving**: Flexibility, spontaneity, and openness
- **Judging**: Organization, planning, and decisiveness

## Behind the scenes

The demo application runs on Heroku using the same Flask architecture as the local version:

### Architecture

```python application.py
from flask import Flask,request,render_template
from predict import classify_user

app = Flask(__name__)

@app.route("/",methods=["GET","POST"])
def home():
    if request.method == 'POST':
        uid = request.form.get("userid")
        name,mbti,personality = classify_user(uid)
        return render_template('analysis.html',display=True,
                             name=name,mbti=mbti,personality=personality)
    return render_template('analysis.html',display=False)
```

### Deployment

The application is deployed on Heroku with:
- **Gunicorn** WSGI server for production
- Pre-trained Naive Bayes models in `Pickle_Data/` directory
- TF-IDF vocabulary file in `CSV_Data/newfrequency300.csv`
- Twitter API credentials configured as environment variables

### Performance

Analysis typically takes 10-30 seconds depending on:
- Number of tweets available (up to 4,000 analyzed)
- Twitter API rate limits
- Text preprocessing complexity
- Server load on Heroku dyno

## Limitations

Be aware of these constraints when using the demo:

- **Protected accounts**: Cannot analyze private Twitter accounts
- **Rate limits**: Twitter API restricts requests per time window
- **Tweet volume**: Accounts with few tweets may produce less accurate results
- **Language**: Optimized for English-language tweets
- **Recency bias**: Only analyzes recent tweets, not entire history
